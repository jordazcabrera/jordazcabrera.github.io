Web-based Distributed Authoring and Versioning (WebDAV)

Both WebDAV and FTP have many similarities like facilitating file transfer and deleting directories. But in addtion to file transfer operations, WebDAV can also do so much more. WebDAV is an extension to HTTP. When we connect to the internet, HTTP lets us view webpages and interact with them by following hyperlinks. We can edit pages if they are shared documents or on a wiki but not when they are read-only. HTTP canâ€™t edit those pages, and so here is where WebDAV comes in. The WebDAV protocol enables a webserver to behave like a fileserver supporting collaborative authoring of web content; editing and file management between users located remotely from each other on the Internet.

History of the WebDAV

  • Tim Berners-Lee created the World Wide Web and the web browser called WorldWideWeb, which could both view and edit web pages. As the web grew, it became ready only medium

  • Members of the World Wide Web Consortium worked togheter in 1996 to overcome that limitation. The meetings resulted in the formation of an IETFworking group, because the new effort would lead to extensions to HTTP.

  • While working on the protocol, it became clear that handling both distributed authoring and versioning together would involve too much work and that the tasks would have to be separated. The WebDAV group focused on distributed authoring.

  • In 2002 an extension for WebDAV was defined.

  • In 2007 WebDAV work concluded with an incremental update approved by IESG.

Advantages

  • Users are just not able to exchange files, they are also able to collaborate over a single when working from different locations.

  • Most operating systems have WedDAV integrated into OS.

  • It makes transferring files similar to using a local netwrok drive.
